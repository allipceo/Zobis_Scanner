# AG_Test02 서비스 시나리오 (Service Scenario) V1.0

## 1. 개요
*   **서비스명**: ZOBIS Name Card Scanner
*   **한줄 요약**: 명함 사진을 찍으면, 알아서 텍스트를 읽고(OCR), 내 구글 주소록에 쏙 넣어주는 비서.

## 2. 사용자 흐름 (User Flow)
1.  **접속**: 웹 브라우저에서 `localhost` (포트 80 또는 프록시 필요) 접속.
2.  **로그인**: "Google 계정으로 로그인" 버튼 클릭 -> OAuth 2.0 인증.
3.  **업로드**:
    *   [명함 촬영/업로드] 버튼 클릭.
    *   카메라로 찍거나 갤러리에서 선택.
    *   **[전처리 로직]**: 이미지를 흑백/고대비로 변환하여 OCR 정확도 향상.
4.  **분석 및 수정 (Validation)**:
    *   화면 좌측: 업로드한 명함 이미지.
    *   화면 우측: OCR로 읽어낸 텍스트 필드 (이름, 전화번호, 이메일, 회사, 직책).
    *   사용자가 잘못 인식된 글자를 수정.
5.  **저장 (Action)**:
    *   [구글 주소록에 저장] 버튼 클릭.
    *   저장 완료 알림 ("조대표님의 연락처가 저장되었습니다!").

## 3. 기술적 제약 및 해결 방안
*   **리다이렉트 URI 이슈**:
    *   Google Cloud Console에 등록된 URI가 `http://localhost` (포트 없음)입니다.
    *   **해결**: 로컬 개발 서버를 **포트 80**으로 띄워야 합니다. (보통 5500이나 3000을 쓰지만, 설정에 맞춰야 함)
    *   *Note*: 윈도우에서 포트 80을 쓰려면 관리자 권한이 필요할 수 있습니다.

## 4. 데이터 흐름 (Data Flow)
`Image` -> `Canvas(Preprocessing)` -> `Tesseract.js(OCR)` -> `Regex Parser` -> `User Review` -> `Google People API` -> `Contacts`

## 5. 승인 필요 사항
*   **포트 80 사용**: `http://localhost`로만 접속해야 로그인이 됩니다. (다른 포트 사용 불가)
