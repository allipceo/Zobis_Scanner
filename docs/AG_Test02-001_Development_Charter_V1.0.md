## 2. 서비스 시나리오 요약 (Scenario)
1.  사용자가 웹 앱에 접속하여 **명함 사진**을 업로드한다.
2.  앱이 이미지에서 **성명, 회사, 직책, 휴대폰, 이메일**을 자동으로 인식한다.
3.  인식된 정보를 사용자가 확인 및 수정하고 **'저장'** 버튼을 누른다.
4.  해당 정보가 **`ceo.allip@gmail.com` 구글 주소록**에 즉시 등록된다.

## 3. 기술적 난이도 분석 (Complexity Analysis)
조대표님의 질문에 대한 안팀장의 기술적 판단입니다.

*   **난이도**: **중상 (Medium-High)** ⭐⭐⭐
*   **이유**:
    1.  **OCR (광학 문자 인식)**: 웹 브라우저에서 한글/영문 명함을 정확히 인식하려면 `Tesseract.js` 같은 무거운 라이브러리를 써야 하며, 이미지 전처리가 필수적입니다.
    2.  **Google API 연동**: 단순 데이터 저장이 아니라, **Google People API** 인증(OAuth 2.0)이 필요합니다. 이는 보안 설정과 토큰 관리가 꽤 까다로운 작업입니다.
    3.  **데이터 파싱**: OCR 결과인 '텍스트 덩어리'에서 이름, 전화번호, 이메일을 정확히 발라내는 정규식(Regex) 로직이 정교해야 합니다.

## 4. 수행 전략 (Strategy)
*   **단계별 접근**:
    *   **1단계 (Core)**: 명함 업로드 -> 텍스트 추출 (OCR 기능 구현)
    *   **2단계 (Validation)**: 추출된 텍스트 파싱 로직 구현
    *   **3단계 (Integration)**: Google People API 연동 및 저장
